# Project Context

- **Owner:** Ricky Kaare Engelharth
- **Project:** Cabazure.Client — a .NET library for generating strongly-typed HTTP clients via C# Source Generators
- **Stack:** C# / .NET 9, netstandard2.0 targets, IIncrementalGenerator, Roslyn, xunit v3, Verify.XUnitV3 + Verify.SourceGenerators for snapshot testing, Atc.Test
- **Created:** 2026-03-10

## Learnings

- Three test projects: `Cabazure.Client.Tests` (generator snapshots), `Cabazure.Client.Runtime.Tests` (runtime), `Cabazure.Client.IntegrationTests` (end-to-end, AOT-compatible)
- Snapshot workflow: test runs → `*.g.received.cs` appears → inspect → rename to `*.g.verified.cs` to accept
- `TestHelper.VerifyEndpoint(...)` is the entry point for all generator snapshot tests
- Diagnostics tests live in `GeneratorDiagnosticsTests.cs` — every validation rule needs coverage here
- Single test: `dotnet test --filter "FullyQualifiedName~{TestMethodName}"`
- Integration tests are AOT-compatible — avoid reflection
- Test methods return `Task` for async verification; `[Fact]` by default, `[Theory]` for parameterized cases

### 2025-01-27: Test Suite Analysis — Coverage Gaps Identified

**Generator Tests (Excellent):**
- 22 snapshot tests covering all HTTP methods, parameter types, pagination, namespaces, nested types
- 8 diagnostic tests for all 7 error codes (ECL001-ECL007)
- 100% feature coverage for basic scenarios

**Critical Gaps Found:**
1. **Runtime attributes** (9 total) — ZERO unit tests; marked `[ExcludeFromCodeCoverage]` but never validated
2. **Response wrappers** (`EndpointResponse<T>`, `PagedResponse<T>`) — NO dedicated tests; status helpers (`IsOk`, `IsNotFound`) untested
3. **Options classes** (`ClientRequestOptions`, `PagedRequestOptions`) — used everywhere but never directly tested
4. **Multiple parameters** (path/query/header) in single endpoint — NOT TESTED; route parsing regex unverified for complex templates like `/items/{id}/details/{detailId}`
5. **URL encoding edge cases** — special chars in path/query parameters NOT TESTED
6. **Extensions** — `HttpRequestMessageExtensions` (SetScopes/GetScopes) and `HttpClientExtensions.WithRequestOptions()` untested
7. **Token expiry buffer** (1-minute margin) and concurrent token requests — NOT TESTED
8. **AOT compatibility** — project marked `<IsAotCompatible>true</IsAotCompatible>` but no CI validation; `[DynamicallyAccessedMembers]` emission not tested

**Test Quality Issues:**
- Two `.g.received.cs` files found (Obsolete_GetEndpoint, UsingAliases) — tests added but snapshots not approved; blocks CI
- No helpers for complex endpoint scenarios; every test writes full interface inline
- No property-based testing; only known scenarios tested

**Pattern Observed:**
Generator output is thoroughly tested (22 snapshots), but runtime library relies on integration tests without unit test coverage. This creates blind spots: if integration tests don't exercise a path (e.g., null CorrelationId, expired token with 59-second buffer), it's untested.

**Recommendation:**
Start with response wrappers & attributes (high user impact), then tackle edge cases (URL encoding, multiple params), finally add property-based testing for generator fuzzing.

### 2026-03-11: Snapshot Cleanup — Orphaned Files Removed

**Task:** Remove unapproved snapshot files blocking CI:
- `CanGenerate_Obsolete_GetEndpoint#TestEndpoint.g.received.cs`
- `CanGenerate_UsingAliases#TestEndpoint.g.received.cs`

**Investigation:**
Searched for corresponding tests in `ClientEndpointGeneratorTests.cs` — neither `CanGenerate_Obsolete_GetEndpoint` nor `CanGenerate_UsingAliases` exist. Only `CanGenerate_Using_Alias` (singular) exists, added in commit a8b622f.

**Root Cause:**
These snapshot files were generated by tests that were never committed, or were renamed/deleted after snapshot generation. The `CanGenerate_UsingAliases` case is a typo/duplicate — the actual test is `CanGenerate_Using_Alias`.

**Action Taken:**
1. Deleted both `.g.received.cs` files (orphaned snapshots)
2. Deleted `CanGenerate_UsingAliases#TestEndpoint.g.verified.cs` (empty file, also orphaned)
3. Confirmed no other `.g.received.cs` files exist in test directory
4. Committed deletion with message explaining orphaned snapshots

**Outcome:**
No unapproved snapshot files remain. CI should pass snapshot verification.

Branch has WIP changes for PATCH support causing build failure (`HttpMethod.Patch` doesn't exist in netstandard2.0), but this is unrelated to snapshot cleanup task.
